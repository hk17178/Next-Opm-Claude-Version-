## 一、全局设计规范

### 1.1 设计语言定位

OpsNexus 采用 **"数据指挥中心"** 设计语言，在传统运维工具表格化界面的基础上，融入粒子网络背景、数据流动光效、扫描线、翻牌交互等高级视觉效果，定位为高端商业化运维产品的视觉标准。

核心原则:
- 数据密度优先: 一屏展示尽可能多的指标，减少翻页和跳转
- 动效传达状态: 粒子网络=系统运行中，流光=数据流动中，脉冲=异常，翻牌=隐藏详情
- 双主题平等: 深色和浅色是两套完整的、平等的设计语言
- 图表大于表格: 列表页必须配套统计分析面板，不允许只有干巴巴的表格

### 1.4 粒子网络背景系统

全局 Canvas 层 (z-index: 0)，所有页面共享同一个粒子实例:

```
粒子: 70个(深色)/65个(浅色), 半径0.3~1.5px, 速度0.15~0.25px/frame
连线: 距离<100px的粒子之间画连线, 透明度=4%(深)/3%(浅)×(1-d/100)
流光: 20条, 拖尾=speed×10px渐变, 绿/蓝各半, 生命周期50~180帧
鼠标: 140px范围内粒子被吸引(force=0.000025), 300px径向光晕跟随
雷达: 右上角(W-50,50), 半径35px, 旋转0.012rad/frame, 2个同心圆
性能: requestAnimationFrame, document.hidden时暂停
降级: hardwareConcurrency<4时粒子减半关流光
```

### 1.5 翻牌效果 (Flip Card)

| 使用位置 | 正面 | 背面 (可自定义) | 时长 |
|---------|------|---------------|:----:|
| 指标概览卡片 (4张) | 当前值+趋势+迷你图 | 环比/同比/峰值/均值/异常次数 | 600ms |
| 健康矩阵色块 | 颜色状态 (绿/黄/红) | CPU/内存/连接数/磁盘 | 500ms |
| 值班人员卡片 | 姓名+角色+状态 | 手机/响应时间/处理数 | 500ms |
| CMDB 资产卡片 | 名称+IP+状态 | 配置摘要 (CPU/内存/OS) | 600ms |

```css
.flip-card { perspective: 800px; }
.flip-inner {
  transition: transform 0.6s cubic-bezier(0.4, 0, 0.2, 1);
  transform-style: preserve-3d;
}
.flip-card:hover .flip-inner { transform: rotateY(180deg); }
.flip-front, .flip-back { backface-visibility: hidden; }
.flip-back { transform: rotateY(180deg); }
```

背面内容由用户自定义: 个人偏好 → 翻牌指标配置，选择每个翻牌位显示哪些字段。

### 1.6 动效规范

| 动效 | 时长 | 场景 |
|------|:----:|------|
| 卡片扫描线 | 4s loop | 指标卡顶部横向光线扫过 |
| 数字翻牌弹入 | 600ms | 页面加载时数字逐位弹入 |
| 实时曲线绘制 | 逐点35ms | 迷你图从左到右逐点画出+末端脉冲点 |
| AI 打字机 | 20~35ms/字 | AI 分析结果逐字输出+闪烁光标 |
| 漏斗流光 | 2s loop | 降噪漏斗每段进度条内白色光点流过 |
| P0 呼吸脉冲 | 3s loop | P0行左侧红色条+健康矩阵红格呼吸 |
| LIVE 指示灯 | 3s loop | 绿色圆点明暗呼吸+box-shadow |
| 主题切换 | 400ms | 所有CSS变量平滑过渡 |
| 翻牌翻转 | 500~600ms | hover触发Y轴180度 |
| 卡片hover抬起 | 350ms | 上移2px+边框提亮 |
| 鼠标光晕 | 实时 | 径向渐变跟随鼠标 |

支持 prefers-reduced-motion: reduce 时关闭所有动画保留静态布局。

## 三、页面设计 (21 页)

原页面 1~10 的 ASCII wireframe 和交互规格保持不变 (详见 v1.0 存档), 增强点:
